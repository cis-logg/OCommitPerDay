CPU : central processor unit
계산과 논리 연산이 이루어지는 중앙처리장치
    구성 : register (레지스터) : 제한된 수의 기억 장소
	고주파수 클럭 : 다른 시스템 구성요소를 CPU의 내부와 동기시킨다.
	CU control unit (제어장치) : 기계 명령어를 실행하는 데 수반되는 단계들의 순서를 조정한다.
	ALU arthmetic logic unit (산술논리연산장치) : 덧셈, 뺄심과 AND, OR, NOT 과 같은 논리 연산을 수행한다.

CPU 
메모리 저장 장치 : 컴퓨터 프로그램이 실행되는 동안 명령어와 데이터를 저장하는 곳
저장 장치는 CPU에서 데이터에 대한 요청을 받아서 RAM에서 CPU로 데이터를 전송하거나 CPU에서 메모리로 데이터를 전송한다.

데이터에 대한 모든 처리는 CPU 내에서 이루어 진다.
메모리에 있는 프로그램은 CPU로 복사되어야 실행할 수 있다.

BUS
데이터를 컴퓨터의 한 부분에서 다른 부분으로 전송하는 병렬 WIRE그룹이다.
데이터 버스 : CPU와 메모리 간에 명령어와 데이터를 전송한다.
입출력 버스 : CPU와 시스템의 입출력 장치 간에 데이터를 전송한다.
제어 버스 : 시스템에 부착된 모든 장치의 동작을 동기화 하기 위해 2진 신호를 사용한다.
주소 버스 : 현제 실행중인 명령어가 CPU와 메모리 간에 데이터를 전송할 때에 명령어와 데이터의 주소를 가지고 있다.

클럭
CPU와 시스템 버스를 포함하는 각 동작은 일정한 속도로 진동하는 내부 클럭에 동기가 된다.
기계어 명령어에 대한 시간의 기본 단위는 기계 사이클 (또는 클럭 사이클)이다.
클럭 사이클 : 한 하강 에지와 다음 하강 에지 사이의 시간
메모리 접근이 있어야 하는 명령어는 CPU와 시스템 버스와 메모리 회로의 속도 차이 때문에 wait state(대기 상태)라고 하는 빈 클럭 사이클을 종종 가진다.


명령어 실행 사이클
(하나의 기계어 명령어 실행)

프로그램은 실행 되기 전에 메모리에 적재된다.
    명령어 포인터 : 다음 명령어의 주소를 포함한다. (IP, program counter)
    명령어 큐 : 실행되려고 하는 명령어들을 저장한다.
    
    기계어 명령어 실행
        fetch : 제어 장치는 명령어 큐에서 다음 명령어를 인출, IP (=program counter) 증가시킨다.
        decode : 제어장치는 명령어의 기능을 해독한다.
                    명령어의 입력 피연산자를 ALU(산술 논리 장치)에 전달
                    수행될 연산을 알려주는 신호를 ALU로 보낸다.
        fatch operand : 명령어가 메모리에 있는 입력 피연산자 사용 시 
                            제어장치는 피연산자를 읽어 내부 레지스터로 복사하기 위해 read operation(읽기 동작) 사용한다.
        excute : ALU는 명명된 레지스터와 내부 레지스터를 피연산자로 사용하여 명령어를 실행한다.
                   출력을 명명된 레지스터나 메모리에 보낸다. 
                   ALU는 프로제서 상태에 대한 정보 제공하는 상태 플래그를 갱신한다.
        store output operand : 출력 피연산자가 메모리에 있다면
                                      제어장치는 데이터를 저장하기 위해 write operation(쓰기 동작) 시작
