입출력 접근 레벨
고급 언어 함수 : C++, JAVA와 같은 고급 프로그래밍 언어에서 제공하는 입출력 함수
	다양한 종류의 컴뷰터 시스템에서 동작, 운영체제에 종속되지 않아 이식성이 있다.
운영체제 : API (application programming interface) 라이브러리에서 운영체제 함수를 호출.
BIOS (basic input output system) : 하드웨어 장치와 직접 통신을 하는 저수준 서브루틴의 집합.
	컴퓨터 제조사가 설치, 컴퓨터의 하드웨어에 알맞게 맞춰진다.

장치 드라이버(device driver) : 운영체제가 하드웨어 장치가 직접 통신 하는 저수준 서브루틴의 집합

HLL hardware loaded library

여러 레벨에서 프로그래밍
Library (레벨3) 
	일반적인 텍스트 입출력과 파일 기반 입출력을 수행, 라이브러리 함수 호출
OS Function (레벨2) 
	일반적인 텍스트 입출력과 파일 기반 I/O를 수행하는 운영체제 함수 호출
BIOS Function (레벨1) 
	색상, 그래픽 등 저수준 디스크 I/O와 같이 디바이스 고유 특징을 제어하는 BIOS 호출.
Hardware (레벨0)
	특정 장치에 대한 완전한 제어권을 가진 하드웨어 포트로부터 데이터를 보내고 받는다.
	다양한 하드에어 장치에서 사용할 수 없으며, 이식성이 없다.
	특정 유형의 장치에 맞도록 프로그램 코드를 작성해야 한다.

레벨 선택 시 고려해야할 교환 조건(tradeoff)
제어권 대 이식성이 주된 교환 조건이다.
OS (레벨2)
	같은 운영체제를 수행하는 어떤 컴퓨터에서든 동작한다.
	각 입출력 호출을 여러 계층 거쳐야 실행되어 특별히 빠르진 않다.
BIOS (레벨1)
	표준 BIOS를 찾춘 모든 시스템에서 동작하지만 모두 같은 결과를 가지진 않음.
	사용자의 하드웨어 설정을 찾아 출력 형식을 맞추는 코드를 작성할 필요가 있음.
	레벨 2보단 빠르다.
Hardware (레벨0)
	직렬 포트와 같은 일반적인 장치와 알려진 제조사에서 만든 입출력 장치에서 동작.
	입출력 장치에서의 여러 변종을 처리하도록 코드 논리를 확장해야함.
