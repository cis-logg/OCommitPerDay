부동 소수점 장치
FPU, floating pint unit : 부동 소수점 장치
	고속 부동소수점 산술 연산을 수행한다.
FPU에는 ST(0), ST(1), ST(2), ST(3), ST(4), ST(5), ST(6), ST(7) 이라는 부동 소수점 데이터 레지스터가 있음.

데이터 버스 크기가 시스템의 성능에 영향을 준다.
	프로세서가 32bit 정수를 메모리로 전송하는 데 8bit 데이터 버스를 사용,
	정수는 4부분으로 분리해야 하고 
	동작을 완성하기 위해 4개의 분리된 데이터 전송 동작이 필요
	8bit 데이터 버스는 32bit 데이터 버스의 1/4 속도로 데이터 전송

인텔 8086 프로세서 (1978)
16bit 레지스터 / 16bit 데이터 버스를 가진다.
프로그램이 1MB의 RAM까지 주소 지정을 할 수 있도록 세그먼트 메모리 모델을 사용

인텔 80286 
보호 모드에서 수행되는 최초의 인텔 프로세서
24bit 주소 버스를 사용, 16MB까지 주소 지정 가능

인텔 80386 프로세서 (1985)
32bit 데이터 레지스터 / 32bit 주소 버스, 외부 데이터 버스 도입
IA-32 계열의 최초의 프로세서
컴퓨터의 물리적 메모리보다 더 큰 가상 메모리의 주소지정 가능
각 프로그램은 4GB의 선형 주소 공간을 부여받는다.

인텔 i486
여러 개의 명령어를 동시에 처리할 수 있는 파이프라이닝 기법을 사용한 명령어 집합 마이크로 구조의 특징을 가짐

팬티엄 프로세서 (1993)
두 개의 병렬 실행 파이프라인이 있는 슈퍼스칼라 포함
두 개의 명령어는 동시에 해독되어 실행할 수 있다.
32bit 주소 버스 / (프로세서 내에 있는) 64bit 내부 데이터 버스를 사용

Inter 64
원래 AMD가 개발한 x86-64 사양을 인텔이 구현
개별 프로세서들이 64bit보다 작은 주소를 구현할지라도 64bit 선형 주소 공간을 제공
물리적 주소 공간은 64GB 보다 더 클 수 있다.
	IA-32e모드
	호환 모드 : 예전의 16bit와 32bit 응용 프로그램을 실행할 수 있다.
	64bit 모드 : 메모리 세그먼테이션이 비활성화 되어 균일 선형주소 공간을 만든다.

하이퍼 쓰래딩과 멀티 코어 처리
dual processor system : 두 개의 독립된 물리적 컴퓨터 프로세서를 포함
	두 개의 독립된 작업(process or thread)이 병렬로 동시에 수행하도록 스케쥴링
intel hyperthreading(HT) : 두 작업을 단일 프로세서에서 동시에 수행
dual core : 같은 직접회로(IC) 패키지에 두 개의 완전한 물리적 컴ㅍ터 프로세서 칩을 포함한 직접회로
	각 프로세서는 자원/프론트 사이드 버스에 대한 독립적인 통신 경로를 갖는다.

CISC Complex Instruction Set Computer, 복잡한 명령어 집합 컴퓨터
microcode라는 언어로 작성된 CPU 내부에 있는 해독기가 기계어 명령어를 해독하고 실행시킨다.

RISC Reduced Instruction Set Computer, 축소 명령어 집합
하드웨어를 사용하여 명령어를 해독하고 실행한다.
