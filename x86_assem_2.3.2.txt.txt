보호 모드
프로그램의 선형 주소 공간 : 4GB (0 ~ FFFFFFFF)

균일(flat) 메모리 모델(.MODEL 디렉티브)
	명령어나 변수의 주소를 보관하기 위해 하나의 32bit 정수만을 필요로 한다.
CPU는 주소 계산과 변환을 백그라운드로 수행, 프로그래머가 알지 못한다.

flat segment (균일 세그먼트 모델)
모든 세그먼트는 컴퓨터의 32bit 물리적 주소 공간 전체에 맵핑된다.
코드/ 데이터 적어도 2개의 세그먼트 필요
각 세그먼트는 GDT (전역 서술자 테이블)에 저장되는 64bit 정수인 세그먼트 서술자로 정의된다.

	base address (베이즈 주소)필드는 사용할 수 있는 메모리의 첫 번째 주소(00000000)를 나타냄
	segment limit (세그먼트 한계) 는 0040
	access (접근)필드는 세그먼트가 어떻게 사용될 수 있는지 결정

다중 세그먼트 모델
각 태스크나 프로그램은 LDT (지역 서술자 테이블)인 자신의 세그먼트 서술자 테이블이 있다.

페이징
세그먼트를 page라고 하는 4096 byte 메모리 블록으로 나눌 수 있게 하는 paging을 지원한다.
virtual memory (가상 메모리) : 운영체제가 맵핑하는 페이지의 전체 집합
virtual memory manager (가상 메모리 관리자) : 운영체제의 유틸리티 프로그램

프로세서가 메모리에서 paging되어 나간 코드를 실행 시 page fault(페이지 오류)를 발생
필요한 코드나 데이터를 포함하는 페이지를 다시 메모리에 적재한다.
